새 기능(특히 SEO·분석·UI 보조 같이 비즈니스 코어가 아닌 항목) 도입을 위해 **Vite·React·React Router·@vitejs/plugin-react·@tailwindcss/vite 등 핵심 의존성의 메이저 업그레이드를 강요하는 옵션은 회피**. 동등 효용을 주는 다른 옵션이 있다면 그쪽을 우선.

**Why:** 2026-04-27 SEO 옵션 2 1단계 보류 후 옵션 A(Vite 7→8 + plugin-react 5→6) vs D(vite-prerender-plugin, 의존성 변경 없음) 선택에서 사용자 명시 발언: "SEO 하나 때문에 다른 핵심적인 라이브러리들의 버전을 변경하게 되면 갑자기 연쇄적으로 근본 시스템에 장애가 생길 것 같은 공포가 든다. 정확히 어떤 문제가 생길지는 모르겠지만." 이는 **불확실성에 대한 합리적 회피** — 메이저 업의 회귀 영역이 광범위해서 사전 예측 불가능, 발견 비용도 높음.

**How to apply:**
- 새 기능 옵션 비교 시 "기존 핵심 의존성 변경량" 칼럼을 트레이드오프 표에 명시
- 핵심 의존성 메이저 업이 필요한 옵션은 명시적 단점으로 강조 ("Vite 8 회귀 위험" 식)
- 동등 효용을 주는 비침투 옵션이 있으면 우선 추천. 없으면 그 사실 자체를 명시
- 핵심 의존성 = 빌드 툴체인(Vite, plugin-react, tailwind), 런타임 코어(React, RR), 상태/데이터(Zustand, RQ). 보조 라이브러리(아이콘·유틸 등)는 해당 안 됨
- 메이저 업이 **다른 독립적 가치**(보안 패치, 신규 기능 다수 활용 등)도 동시에 가져오면 별도 결정으로 분리해 평가
