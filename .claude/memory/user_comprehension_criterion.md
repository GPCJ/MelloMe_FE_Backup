사용자의 지식 흡수 판단 기준은 **"머리 속에 이미지/구조가 떠오르는가"**. 텍스트를 읽고 이해한 느낌만 있고 구조가 안 떠오르면 아직 흡수되지 않은 것으로 본인이 명시적으로 분류함 (2026-04-21, resolveImageUrl 설명 리뷰 중).

예: "new URL은 이러이러한 역할" 을 문장으로 설명받았을 때 — 문장은 따라가지만 URL 인스턴스가 "칸이 나뉜 양식지" 로 그려질 때까지 이해가 덜 완성됐다고 판단.

**How to use:**
- 개념 설명 시 텍스트 나열에서 멈추지 말고, 다음 중 하나 이상으로 **시각 구조 트리거** 제공:
  - 비유 (주소 만드는 사람 vs 택배기사, 양식지 채우는 기계 등)
  - ASCII 다이어그램 (플로우, 계층, 관계 표시)
  - 엣지 케이스 표 (입력 → 출력 매핑)
  - 단계별 추적 (입력이 코드 안에서 어떻게 흐르는지)
- 사용자가 "아직 이미지 안 떠오름" 이나 "텍스트로만 이해됐다" 신호를 주면:
  - 같은 설명을 반복하지 말고 **다른 시각화 각도** 시도 (구조도 → 비유, 비유 → 표, 표 → 추적 등)
  - 어느 단계가 특히 흐릿한지 좁혀서 질문
- 관련 메모리: wiki `19` (프론트 학습 정리), `user_knowledge_auth` (비유적 설명 선호)
