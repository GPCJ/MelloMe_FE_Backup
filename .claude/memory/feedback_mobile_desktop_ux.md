---

모바일 디자인을 받아 구현할 때 데스크탑 UX도 함께 설계할 것.

**Why:** 댓글 UI 작업 시 모바일 전용 페이지로만 구현했다가 데스크탑에서는 인라인 input이 필요하다는 추가 요청이 발생. 처음부터 양쪽을 고려했으면 재작업을 줄일 수 있었음.

**How to apply:** 모바일 디자인을 받으면 "데스크탑에서는 어떻게 동작할지"를 구현 전에 먼저 확인. 이 프로젝트의 기본 패턴: 모바일은 페이지 전환 UX, 데스크탑은 인라인 UX로 분기 (Tailwind `md:` 브레이크포인트 활용).
