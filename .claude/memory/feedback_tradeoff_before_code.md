description: 모든 코드 수정/삭제/생성 작업 시 트레이드오프를 먼저 짚고 넘어가야 함 — 사용자의 기술적 판단력 성장 목적
type: feedback
---

코드 작업(수정, 삭제, 생성) 실행 전에 반드시 해당 작업의 트레이드오프를 설명하고 넘어갈 것.

**Why:** 사용자가 AI 도구에 코드 작성을 의존하면서 "이 코드가 왜 이렇게 되어야 하는지", "다른 방법 대비 어떤 장단점이 있는지"를 직접 고민하는 경험이 부족해지고 있음. 면접에서 특정 코드에 대해 깊이 있는 기술 대화를 할 수 있는 역량을 기르고 싶어함.

**How to apply:**
- 코드 작업 계획 단계에서 (혹은 계획 없이 바로 진행하더라도) 작업 실행 전에 트레이드오프를 짧게 짚어줄 것
- 예: "이 방식은 A 장점이 있지만 B 트레이드오프가 있고, 현재 프로젝트 규모에서는 감당 가능함"
- 대안이 있다면 왜 이 방식을 선택했는지 근거 포함
- 사용자가 스스로 기술적 판단을 내릴 수 있도록 돕는 것이 목적
- **우선순위 높음** — 매 코드 작업마다 적용
