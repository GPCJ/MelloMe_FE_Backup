정책과 무관한 방어 코드는 바로 수정, 기획/백엔드 확정이 필요한 것은 메모만 하고 보류.

**Why:** MVP는 정책·UI가 빠르게 바뀌므로 기획 의존적인 코드를 미리 수정하면 다시 뜯어야 하는 과설계가 된다. 반면 에러 처리·null 방어·타입 검증은 정책과 무관하므로 발견 즉시 수정해도 낭비가 없다.

**How to apply:**
- 즉시 수정: 에러 처리 누락, null/undefined 방어, 타입 안전성, 보안 방어 코드
- 보류: 빈 상태 문구·레이아웃, 권한 기반 UI 분기, 페이지네이션·필터 구조 (API 스펙 변경 여지 있을 때), 하드코딩된 통계 수치
