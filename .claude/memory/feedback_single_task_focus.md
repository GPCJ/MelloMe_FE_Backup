사용자가 특정 작업에 진입했을 때, 해당 작업 맥락만 다룬다. 인접 TODO·미래 제안·겸사겸사 정리는 현재 작업 흐름을 끊으므로 금지.

**Why:** 사용자는 싱글태스크 선호형이며, 작업 중 다른 생각이 스쳐서 집중 유지가 어렵다는 자각을 갖고 있음 (2026-04-22 본인 명시). 멀티 정보 덤핑이 이 집중 흐름을 깬다. 본인이 개선 의지를 표명하며 "3초 캡처 → notepad → 즉시 복귀" 루틴을 수락함 → `user_work_focus_pattern.md`

**How to apply:**
- 작업 진행 중 답변은 현재 작업 맥락에만 한정. "겸사겸사", "다음으로", "이 김에", "추가로 X도" 류의 곁가지 제안 자제
- 작업 중 떠오른 새 아이디어·발견한 TODO는 notepad(`priority` 또는 `working`)에 즉시 기록하고 현재 작업으로 복귀 유도. 대화 본문에 나열하지 않음
- 단, 현재 작업 수행에 필수적인 의존 정보(예: 해당 작업 성공 기준·블로킹 요인)는 제시 — 곁가지가 아니므로
- 예외: 사용자가 명시적으로 "다른 것도 정리해줘"·"전체 상황 보자"고 요청한 경우에만 맥락 확장
