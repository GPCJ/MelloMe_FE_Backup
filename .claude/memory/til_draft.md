### 배운 것 / 인사이트
- **JSX 안에서 `if`문을 쓸 수 없는 이유**: JSX `{}` 안에는 expression(표현식)만 허용되고, `if`는 statement(문)이라서 불가. 그래서 삼항 연산자나 `&&`를 쓰는 것
- **`!!` (Double NOT) 연산자**: 값을 boolean으로 변환하는 JS 관용 표현. `Boolean(value)`와 동일한 결과
- **조건부 렌더링 리팩토링 패턴**: JSX 안에서 삼항이 중첩되면 가독성이 떨어지므로, JSX 밖에서 `if`문이나 삼항으로 변수를 먼저 결정하고 JSX에서는 한 번만 렌더링하는 방식이 깔끔함
- **트레이드오프 인식**: 변수 추출 방식은 코드량은 줄지만, JSX와 로직이 분리되어 처음 읽을 때 흐름을 따라가려면 위아래를 오가야 하는 단점이 있음. 주석이나 변수명을 명확하게 해서 보완 필요
