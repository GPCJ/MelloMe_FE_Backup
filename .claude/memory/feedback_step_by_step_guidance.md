큰 작업(여러 파일 변경, hook 설계, 마이그레이션 등)을 가이드할 때 한꺼번에 전체 절차/체크리스트를 덤프하지 말 것. 한 단계 가이드 → 사용자 작성 → 검증 → 다음 단계 형태로 잘게 끊어 진행.

**Why:** 2026-05-04 댓글 리액션 hook 작업 중 사용자가 "작업 단위를 나눠서 하나씩 진행하자. 한번에 모든 절차를 설명하지 말아줘"라고 명시. 직접 코딩 모드 + 학습 모드일 때 인지 부하가 크고, 통합 절차를 한꺼번에 받으면 어디서부터 손대야 할지 흐려져 학습 효과↓ + 진행 속도↓. 한 단계씩 진행하면 사용자가 그림을 차곡차곡 쌓아갈 수 있음.

**How to apply:**

- 작업 단위(Task)를 5~7단계로 잘게 쪼개고, 한 번에 1단계만 가이드 (`Step A`, `Step B`, ...)
- 한 단계가 끝나야 다음 단계 가이드 제시 — 미리 다음 단계 절차 흘리지 말 것
- 검증 사이클: 사용자가 코드 보여주면 → 짚을 점 진단 → 작은 수정 → 다음 단계
- 한 단계 안에서도 진행이 막히면 더 잘게 쪼갬 (예: 낙관 업데이트 블록 → B-1 셋업 / B-2 카운트 갱신 / B-3 myReactionType 갱신 식)
- 단, 사용자가 "전체 흐름 한 번 보여줘" 명시 요청 시 예외 — 그땐 통합 가이드 제공 후 다시 단계로 복귀
