AI가 설계·작성을 주도한 작업 뒤 `/wrap-up` 들어가기 전에, **소크라테스식 6~8문항 Q&A**로 이해도 진단을 **기본 제안**으로 언급.

**Why:** 결과물 완료 ≠ 학습 완료. 2026-04-23 GA4/Clarity 설치 후 이 방식으로 "robots vs sitemap / OG unfurling 주체 / 큐잉 버퍼 패턴" 3개 구멍을 정확히 잡아냈음. 사용자 기준(이미지/구조 떠올라야 흡수)과도 맞물림.

**How to apply:** 아래 트리거 조합(A+C)으로 판단. 특정 사용자 표현에 매이지 않음.

---

## 트리거 방식 (A + C 조합, 기준 50%)

### ① 작업 시작 시 AI가 모드 선언 (C)

AI가 코드 작업 계획 공유 직후 한 줄로 예상 모드 선언:
```
이번 작업 예상 모드: [ 사용자 주도 / 혼합 / AI 주도 ]
```

- **사용자 주도**: 사용자가 대부분 타이핑, AI는 힌트/리뷰만
- **혼합**: 설계 논의는 함께, 타이핑 분담
- **AI 주도**: AI가 설계+작성 모두 담당

### ② 작업 종료 시 간단 수치 공유 (A)

커밋 직전 라인 비율 1줄 공유:
```
최종: AI XX줄 / 사용자 YY줄 = AI NN% (트리거 기준 50%)
```

### ③ 트리거 조건

- **AI 주도 선언** → 자동 트리거
- **혼합 선언 + 실제 AI 50%+** → 트리거
- **사용자 주도 선언** → 트리거 스킵 (단, 사용자가 원하면 언제든 요청 가능)

### ④ Q&A 방식

- 6~8문항을 **한 번에 전부** 던짐 (사용자가 리듬 잡기 편함)
- 사용자는 각 문항에 **한 줄로** 답하거나 "모름"이라고 응답
- AI는 답변 모두 받은 뒤 **틀린 것·빈 것만** 보강 (정답인 건 짧게 확인만)
- 난이도 혼합: 쉬운 것(자신감) + 중간(연결) + 어려운 것(구조/메커니즘)
- 보강 시 비유/그림/구조 도식 활용 (사용자 흡수 기준)

### ⑤ Q&A 후 흐름

- `/wrap-up` 진행
- 노션 초안에 **"이해한 것 / 아직 구멍인 것"** 섹션 포함 → 다음 재학습 기준점

---

## 예외 처리

- 라인 비율이 50% 미만이어도 **설계·구조·패턴을 AI가 주도했다면** 트리거 (코드 양보다 개념 위임이 인지부채 본체)
- 단순 CRUD/설정/포매팅 작업은 학습 가치 낮아 트리거 스킵
- 사용자가 이미 충분히 설명받으며 타이핑한 작업은 스킵 가능 (사용자가 판단)
