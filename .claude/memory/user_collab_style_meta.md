사용자는 자신의 작업 프로세스 약점을 투명하게 드러내고, 이를 구조적으로 막기 위한 **AI 쪽 행동 규약을 함께 설계**하는 메타 협업 스타일.

자각하는 본인 약점:
- 피곤하거나 대충 작업할 때 "ㅇㅇ"/"그래"로 자동 승인해버림
- AI 제안을 검증 없이 따라가면 삽질이 누적됨
- 틀린 진단에도 그럴듯한 설명이면 쉽게 승인됨

대응 패턴 (2026-04-21 세션에서 관찰):
1. 실수 한 번 발견 → 원인 분석 → 재발 방지 규약을 AI에게 정의해달라고 요구
2. AI가 초안을 내면 그 한계까지 스스로 짚음 ("그래도 X가 걱정되는데")
3. 보완 조항을 추가로 요구 (경쟁 가설 필수, 궁금증 훅 도입 등)
4. 최종 규약을 feedback 메모리로 저장해 다음 세션에서 강제 적용

**How to apply:**
- 같은 실수가 2회 이상 반복되면 즉시 **"이걸 규칙으로 만들까요?"** 제안. 거부감 없음
- 메타 규칙 논의는 시간 낭비가 아니라 사용자가 가치 있게 여기는 활동. 회피하지 말 것
- 규칙 초안을 낼 때는 **한계/빈틈까지 스스로 드러내기** — 사용자가 추가 보완을 설계할 수 있게
- `feedback_explain_before_act.md`, `feedback_verify_spec_before_workaround.md`가 이 스타일로 공동 설계된 첫 결과물

**Why:** 2026-04-21 PATCH /me workaround 건에서 사용자가 "내가 직접 코딩했다면 이런 일 없었을까?"라며 반성 → 그 자리에서 재발 방지 규약을 바로 메모리로 만들자고 주도. 이런 방식이 몇 차례 반복되면서 패턴화 확인됨.
