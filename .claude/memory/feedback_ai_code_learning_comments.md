AI가 작성/리팩토링한 비자명한 코드는 push 전에 **교육용 밀도 높은 주석**을 먼저 요청해서 리뷰한 뒤 승인한다. 주석의 밀도는 "왜/동작원리/비유"까지 포함 가능하며, 이해가 굳어진 뒤엔 본 코드 주석을 슬림하게 줄일지 그대로 둘지 본인이 결정한다.

**Why:**
- AI 작성 코드는 인지부채 위험 HIGH (project_ai_written_code_cognitive_debt 연계)
- 학습 단계를 "주석 리뷰" 로 강제 삽입하면 `코드를 내 것으로 만드는` 장치가 됨
- 2026-04-21 resolveImageUrl T3 리팩토링 때 실제로 이 방식을 사용해 "머리속 이미지" 가 잡힐 때까지 주석을 읽고 승인 → 본인 스타일로 확립

**How to apply:**
- 리팩토링/새 로직/비자명한 유틸을 AI가 작성하면 push 전 "이해 체크 필요?" 짧게 확인
- 사용자가 학습 의도 밝히면: (1) 코드 변경은 이미 커밋 (2) 그 위에 교육용 주석을 별도 커밋으로 얹고 (3) 사용자 리뷰/질문 → 승인 → push
- 주석엔 함수 계약, 분기 규칙, 비유, 엣지 케이스 표, IIFE 같은 패턴의 "왜" 포함
- 리뷰 중 사용자가 "이미지 안 떠오름" 신호 주면 `user_comprehension_criterion` 적용해 비유/ASCII 구조로 다시 설명
- 승인 후엔 주석을 슬림화할지 그대로 둘지 사용자에게 확인 (기본은 그대로 유지, 짐이 되면 정리)
