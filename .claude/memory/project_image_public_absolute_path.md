백엔드에서 이미지 관련 두 가지 수정 완료했다고 알림 (04-06):
1. 이미지 조회를 public하게 변경 (인증 불필요)
2. 이미지 경로를 절대 경로로 응답

**Why:** #6 이슈에서 요청한 사항 — `<img>` 태그가 Authorization 헤더를 보낼 수 없어서 인증 필수 이미지는 표시 불가했음
**How to apply:** 프론트 검증 아직 미완료. 브라우저 Network 탭에서 응답의 이미지 경로가 `https://...` 절대 경로인지, 시크릿 모드에서 토큰 없이 이미지 접근 가능한지 확인 필요. `resolveImageUrl()` 유틸은 이미 절대 경로 대응 가능 (http로 시작하면 그대로 반환).
