한국어 wiki 페이지, 의사결정 기록, 포트폴리오용 장문 문서는 `~합니다` / `~입니다` 존댓말로 통일한다. **본인용(개발자용) 문서도 동일하게 존댓말로 쓴다**. `~한다` 해라체 혼재 금지.

**예외 (원형 유지)**
- 인용부 안의 원문: `"분석 쿠키를 수집한다"는 조항` 같은 경우 따옴표 내부는 원형
- 의문형 종결 `~인가.` `~는가.` 도 그대로
- 코드 블록 내부 주석·문자열

**Why:** 2026-04-24 사용자가 Claude 작성 문서(ga4-clarity-pm.md, ga4-clarity-dev.md) 두 개 모두 말투를 `~한다` → `~합니다`로 일괄 전환 요청. 포트폴리오/팀 공유 가능성을 고려한 일관된 톤 선호. 본인용 문서도 동일 적용했다는 것이 특이점 — "누가 읽든 같은 톤이 낫다"는 판단.

**How to apply:**
- `wiki_add` / `wiki_ingest` 등으로 장문 문서 생성 시 기본 존댓말
- 기존 `~한다` 문서 발견 시 사용자에게 일괄 변환 의사 확인 권장
- 변환 작업 시 인용부·의문형은 건드리지 말 것 (사용자가 명시)
- 관련 메모리: `feedback_ai_prose_patterns.md`, `feedback_audience_split_docs.md`
