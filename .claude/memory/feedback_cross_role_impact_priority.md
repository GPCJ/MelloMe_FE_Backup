데일리 작업 후보가 여럿일 때, **내 작업물이 다른 직군에게 넘어가야 하는 것을 먼저** 시작합니다.

**Why:** 내 작업이 PM 법적 검토, 디자이너 UI 컨펌, 백엔드 스펙 확정, 법무 검토 같은 타 직군의 후속 작업을 트리거하면, 내가 넘긴 시점부터 상대방의 대기 시간이 시작됩니다. 자기완결(프론트 단독 완결) 작업을 먼저 처리하면 내가 끝낸 뒤에야 상대 쪽 작업이 시작되어 전체 리드타임이 길어집니다. 타인 의존 작업을 먼저 던져두면 "내가 다음 작업 하는 동안 상대방도 병렬로 검토"가 가능합니다.

**How to apply:**
- "오늘 뭐하지?" / "다음 작업 뭐가 좋을까?" 류 질문에 복수 후보가 있을 때, 타 직군 후속 작업을 트리거하는 것을 1순위로 제안합니다
- 트리거 예시: PM 법적 검토 요청, 디자이너 피그마 피드백, 백엔드 스펙 확정, 법무/운영자 컨펌, 지원팀 공지 수정
- 자기완결 작업(순수 프론트 리팩토링, 내부 학습 태스크)은 그 뒤 순서로 미룹니다
- 2026-04-24 사례: 유저 행동 데이터 수집 남은 3개 중 "개인정보처리방침 문구 업데이트 → PM 법적 검토 필요"를 1순위로 선택, "이벤트 4개 코드 삽입(프론트 단독)"을 후순위로 배치
