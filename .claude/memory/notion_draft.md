# 면접 준비 — 새 페이지 생성 (2026-04-09)

**작업:** "면접 준비" 부모 페이지를 새로 생성하고, 아래 내용을 첫 번째 서브 페이지로 추가

## 서브 페이지: 커밋 히스토리가 왜 지저분한가요?

**Q. 커밋 히스토리를 보면 자동 커밋이 많은데, 왜 이렇게 관리했나요?**

MVP 개발 초기에는 기능 구현 속도를 최우선으로 두고 빠른 프로토타이핑에 집중했습니다. 두 개발 환경(맥북/WSL2)을 오가며 작업하다 보니, 환경 간 코드 동기화를 위한 자동 백업 스크립트를 사용했고 그 과정에서 동기화용 커밋이 다수 생성되었습니다.

핵심 기능이 어느 정도 완성된 시점에서 커밋 히스토리의 가독성 문제를 인지했고, 이후부터는 기능 단위 커밋 + conventional commit 컨벤션으로 전환했습니다.

**핵심:** 몰라서가 아니라 의식적으로 속도에 우선순위를 둔 결과이며, 문제를 인지한 뒤 스스로 개선했습니다.
