## 주간 기록 (2026-03-31 ~ 04-06) 에 추가할 항목

**04-03 | 프론트엔드 아키텍처 점검 (Before)**

백엔드·디자이너 작업 대기 중, 프론트 코드의 현재 상태를 아키텍처 관점에서 점검했습니다.

**점검 결과 요약:**
| 항목 | 현재 상태 | 비고 |
|------|----------|------|
| 폴더 구조 | MVP에 적절 | 기능 늘어나면 도메인별 분리 필요 |
| 상태 관리 | 인증만 전역, 나머지 로컬 — 적절 | |
| API 연동 구조 | 잘 분리됨 | 토큰 갱신 로직 소폭 개선 여지 |
| 타입 안전성 | 도메인별 타입 파일 분리, 충실 | |
| 라우팅/접근 제어 | 3단계 보호 (비회원/회원/치료사) 잘 구성 | |
| 코드 응집도 | 페이지 파일에 로직이 집중됨 | 기능 추가 시 분리 필요 |

**주요 개선 포인트 (After에서 적용 예정):**
1. 페이지 파일의 데이터 로직을 별도 훅으로 분리 (예: 게시글 목록 조회, 댓글 관리)
2. 토큰 갱신 시 안전하지 않은 호출 방식 개선
3. 뒤로가기 버튼이 외부 유입 시 오동작하는 문제 수정
4. 임시 목데이터(공지사항 등) 정리

**결론:** 현재 MVP 규모에는 적절한 구조. "지금 당장 문제"보다는 "기능 추가 전에 정리하면 좋을 것들" 수준.
