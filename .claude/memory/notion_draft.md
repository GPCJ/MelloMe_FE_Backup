## 4월 7일 작업 내역

### 1. 폴더 구조 리팩토링
- `components/`와 `pages/`를 도메인별 하위 폴더로 재구성 (auth, post, common, layout 등)
- 파일이 많아지면서 관련 파일 찾기 어려운 문제 해결
- 모든 import 경로 업데이트 완료, 빌드 검증 완료

> **어필 포인트:** flat 구조에서 파일 수가 늘어나며 발생한 DX(개발자 경험) 저하를 인지하고, 도메인 기반 폴더 구조로 선제 리팩토링. 기능 확장 시 관련 파일이 같은 폴더에 모이도록 설계해 협업 효율성과 온보딩 용이성을 높임. 전체 import 경로 마이그레이션 후 tsc 빌드 검증까지 완료하여 무중단 리팩토링 수행.

### 2. 댓글 UI 리디자인
- 디자이너 모바일 디자인 기반으로 댓글 시스템 3페이지 구조 구현
  - 게시글 상세 → 댓글 목록 표시
  - 댓글 작성 페이지 (모바일 전용 진입)
  - 댓글 상세 → 대댓글 목록 + 작성
- 모바일/데스크탑 UX 분기 적용
- CommentCard 공통 컴포넌트 생성 (리액션 아이콘 UI)

> **어필 포인트:** 디자이너의 모바일 퍼스트 시안을 받아 모바일과 데스크탑에서 서로 다른 UX 흐름을 설계·구현. 모바일은 페이지 전환 기반(앱스러운 경험), 데스크탑은 인라인 입력 기반으로 분기 처리. 향후 PWA/Capacitor 전환 시 네이티브 앱과 유사한 네비게이션을 제공하기 위한 구조적 기반을 미리 마련.

### 3. 공통 컴포넌트 추출 (코드 정리)
- **Pagination** — 검색/게시글 목록에서 중복되던 페이지네이션 UI를 하나의 공통 컴포넌트로 분리
- **FilterChips** — 치료 분야 필터 칩(전체/작업치료/언어치료 등) UI를 공통 컴포넌트로 분리
- 두 컴포넌트 모두 `components/common/`에 배치

> **어필 포인트:** 코드 리뷰 과정에서 2개 페이지에 걸친 중복 UI 패턴을 식별하고, props 인터페이스(`value`, `onChange`)를 설계하여 공통 컴포넌트로 추출. 기존에 `constants/post.ts`에 존재하던 유사 상수(`THERAPY_CHIPS`)와 용도를 구분하여 필터링용 `FILTER_CHIPS`를 별도 정의하는 등 데이터와 UI의 관심사를 분리. 스타일 불일치(`py-1.5` vs `py-2`)도 통일.

### 4. 검색 기능 코드 리뷰
- SearchPage 코드 점검 후 8개 이슈 발견 및 개선 전략 수립
- 즉시 실행 항목(공통 컴포넌트 추출) 모두 완료
- 나머지(URL 파라미터 통합, 커스텀 훅 분리 등)는 MVP 이후 진행 예정

> **어필 포인트:** 자체 코드 리뷰를 통해 race condition(AbortController 부재), stale closure 위험, URL/state 이중 관리 등 8개 이슈를 분류(버그/잠재적 문제/UX)하고, MVP 일정과 코드 품질 사이의 트레이드오프를 고려한 점진적 개선 전략(C+A 방안) 수립. "공통 컴포넌트 먼저 추출 → 새 코드부터 적용 → MVP 이후 기존 코드 리팩토링" 순서로 우선순위를 나누어 실행.
