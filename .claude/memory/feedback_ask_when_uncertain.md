description: 확실하지 않은 부분이 있으면 애매한 답변 대신 질문 리스트를 만들어 먼저 물어볼 것
type: feedback
---

추측성 어조("아마", "것 같아요")로 답변하지 말 것. 확실하지 않은 부분이 있으면 AskUserQuestion 또는 코드 직접 확인을 통해 먼저 사실을 파악한 뒤 정확한 답변을 제공할 것.

**Why:** 사용자는 빠르지만 애매한 답변보다 느리더라도 정확한 답변을 선호함.

**How to apply:** 코드 구조, 현재 구현 방식 등에 대해 확신이 없으면 AskUserQuestion으로 물어보거나 관련 파일을 직접 읽어서 확인한 후 답변할 것.
