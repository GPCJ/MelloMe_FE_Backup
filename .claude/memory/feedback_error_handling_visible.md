description: .catch(() => {})로 에러 무시하지 말고, QA 친화적 에러 메시지를 보여줘야 함
type: feedback
---

에러를 삼키는 코드(.catch(() => {}))를 작성하지 말 것. 실패 시 사용자에게 알려야 함.

**Why:** MVP QA 단계에서 에러가 보이지 않으면 문제를 발견할 수 없음. jin24님이 직접 UX 관점에서 지적.

**How to apply:** API 호출 실패 시 alert 또는 UI로 에러 표시. 문구는 "실패했습니다. 다시 시도해주세요. (네트워크 탭 참조)" 처럼 개발자 친화적 + 비개발자도 문제 인지 가능한 수준으로.
