권장 옵션을 고를 때 메모리에 저장된 규칙(예: `feedback_single_task_focus`, `feedback_concise_when_tired`)을 반사적으로 적용하지 말 것. **사용자 현재 상태(피로도, 집중력, 다른 마감)를 먼저 추정하거나 짧게 확인한 뒤** 그 상태에 맞는 권장을 제시.

**Why:** 2026-04-27 SEO 1단계 보류 후 "재검토 시점" 질문에서, 사용자가 같은 세션 맥락에 머물러 있고 집중력 OK였는데도 `feedback_single_task_focus`를 자동 적용해 "다음 세션으로 미루자"를 추천. 사용자 지적: 판단은 옳았지만(1번 추천이 맞음) **컨디션 미고려 + 매뉴얼 맹목 적용**이 원인.

**How to apply:**
- 옵션을 추천할 때, 추천 자체에 "**현재 상태에 따라 분기**" 형식으로 사용자 컨디션 변수 노출(예: "집중력 OK → 1, 떨어졌으면 → 2")
- 컨디션이 불명확하면 한 줄로 짧게 묻기. 추측하지 말 것
- 같은 작업 체인 안의 후속 단계는 "곁가지"가 아니라 "단일 작업 연장" — `feedback_single_task_focus` 적용 대상 아님 (곁가지 차단 규칙이지 작업 완료 차단 규칙이 아님)
- 관련 규칙: `feedback_verify_rules_root_cause`(규칙 맹목 적용 금지)와 결이 같으나 축이 다름 — 이쪽은 "사용자 컨디션 축", 그쪽은 "원인 진단 축"
