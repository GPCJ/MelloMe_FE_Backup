API 항목 검토 시 "그때그때 수정" 방식으로 진행할 것 (도메인 단위 묶기 or 전체 보고 한번에 수정 X).

**Why:** 광범위한 수정은 누락 발생 쉽고, 여러 맥락을 참작한 작업은 AI가 약함. 항목별로 바로 수정하면 맥락이 신선하고 검증도 즉시 가능.

**How to apply:** CSV 항목 하나 검토 완료 → 수정 필요하면 바로 수정 → tsc 확인 → 다음 항목으로.
