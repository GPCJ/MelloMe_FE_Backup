비슷한 UX 설계 논의가 3~4개 모이면 "PM/PD 협업 UX 설계 경험"으로 묶어서 빌더스 리그 TIL에 정리 제안할 것.

**Why:** 개별 건은 임팩트가 약하지만, 묶으면 포트폴리오 어필 가능
**How to apply:** UX 플로우 관련 팀 논의가 있을 때마다 여기에 추가 → 충분히 쌓이면 제안

### 2. 게시글 에디터 방향 (2026-03-30, 미확정)
- **선택지**: tip tap 리치텍스트 에디터 vs X/Threads 스타일 심플 피드
- **결정**: 미확정 — 모바일 앱 친화적으로 가려는 방향으로 논의 중
- **영향 범위**: 게시글 작성 UI 전체 + 에디터 라이브러리 선택
- **고심 포인트**: "마크업/서식 지원 vs 가볍고 빠른 피드" 서비스 성격 결정 문제
- **대기**: PM/디자이너 확정 필요. 확정 전 에디터 작업 시작 금지

### 1. 회원가입 후 UI 흐름 (2026-03-26)
- **선택지**: 회원가입 → 바로 인증 (직행) vs 회원가입 → 환영 → 인증 → 축하 → 커뮤니티
- **결정**: PM/PD 제안(환영 페이지 포함) 채택
- **이유**: 향후 관리자 승인 대기 플로우 생길 때 환영 페이지가 상태 안내 역할로 확장 가능. MVP 소수 테스트라 약간의 UX 불편 허용 가능
- **고심 포인트**: "지금의 간결함 vs 나중의 확장성" 트레이드오프
