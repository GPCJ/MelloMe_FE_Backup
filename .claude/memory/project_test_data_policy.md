type: project

테스트 데이터(게시글 등) 삽입은 백엔드에 요청해서 처리. 프론트에서 콘솔 스크립트 등으로 직접 넣지 않음.

**Why:** 프론트가 직접 넣으면 백엔드 DB 구조나 제약 조건을 정확히 모르는 상태에서 작업하게 되어 이슈 발생 위험 있음. 테스트 데이터는 백엔드가 책임지는 영역.

**How to apply:** 전체 기능 테스트가 필요할 때 "테스트 데이터 넣어달라"고 백엔드에 요청하고, 데이터 준비되면 API 동작 확인하는 순서로 진행.
