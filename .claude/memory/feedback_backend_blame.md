type: feedback
---

서버와 통신 중 에러가 발생하면, 백엔드에 부탁하기 전에 프론트에서 시도할 수 있는 모든 방법을 먼저 알려줄 것. 그래도 안 될 때만 백엔드 이슈로 넘길 것.

**Why:** 백엔드가 이미 수정했다고 했는데도 CORS 에러가 계속 나는 상황에서 무조건 백엔드 탓만 해서 답답했음.

**How to apply:** 네트워크/통신 에러 발생 시 — 요청 URL 확인, 환경변수 확인, 재배포 확인, vercel.json 설정 확인 등 프론트 체크리스트 먼저. 백엔드 설정 요청은 마지막 수단.
