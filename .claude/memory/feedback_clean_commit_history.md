---

매 작업이 하나의 의미 있는 단위로 완료되면 "커밋 한번 할까요?" 라고 물어볼 것.

**Why:** 포트폴리오 레포(airo)의 커밋 히스토리가 자동 동기화 커밋으로 지저분해짐. 면접관이 봤을 때 개발 흐름이 보이는 클린한 히스토리 필요.

**How to apply:**
- 기능 구현, 리팩토링, 버그 수정 등 의미 있는 작업 완료 시 커밋 제안
- 커밋 메시지는 feat/fix/refactor 등 conventional commit 형식
- 자동 동기화(push-mello)와 실제 코드 커밋을 분리해서 인식
- Co-Authored-By는 사용자 판단에 맡기기
