/session-bridge 실행 시, 단순히 다음 작업 목록만 저장하는 게 아니라 아래 기준에 해당하는 내용이 있으면 장기 메모리(project/feedback)로 저장할지 사용자에게 별도로 물어볼 것.

저장 제안 기준:
- **정책/결정 사항** — 팀 회의에서 나온 방향 결정, 기술 방식 확정, 플로우 변경 등
- **코드 변경 규모가 큰 작업** — 여러 파일에 걸친 수정, 파일 삭제/추가, 구조 변경 등
- **토큰 소모가 많은 작업** — 복잡한 구현, 긴 디버깅 과정, 설계 논의 등

**Why:** 세션 브릿지는 일회성이라 다음 /session-bridge 실행 시 삭제됨. 중요한 결정이나 큰 작업이 기록 없이 사라지는 걸 방지하기 위함.

**How to apply:** /session-bridge Step 2(현재 상태 파악) 이후, 위 기준에 해당하는 항목을 별도로 나열하고 "이 내용들을 장기 메모리로 저장할까요?" 라고 물어보기. 사용자가 승인하면 적절한 project/feedback 메모리 파일로 저장.
