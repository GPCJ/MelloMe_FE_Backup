코드 수정 시 반드시 아래 순서를 따를 것:

1. **grep 키워드 선정** — 변경되는 함수명/타입명/필드명 중 다른 파일에서 참조할 가능성 있는 것 추출
2. **grep 실행** — 영향 받는 파일/라인 전수 확인
3. **수정 계획 보고** — 어떤 파일의 몇 번 줄을 어떻게 바꿀지 목록으로 제시
4. **승인 대기**
5. **수정 실행**
6. **`npx tsc -b`** 확인

**Why:** setAuth 시그니처 변경 시 grep 없이 수정했다가 의존 파일 5개 누락 → tsc 에러로 뒤늦게 발견. grep이 선행되어야 tsc도 의미 있음.

**How to apply:** 함수/타입/필드명 변경이 포함된 모든 수정에 적용. tsc만 믿지 말 것 — 런타임 의존성은 grep으로만 잡힘.
