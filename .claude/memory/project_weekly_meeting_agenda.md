## 안건 1: "로그인 상태 유지" 체크박스 정책

현재 Refresh Token 7일 고정인데, 체크박스가 의미를 가지려면 체크 여부에 따라 RT 만료 시간이 달라져야 함.

**선택지:**
1. 체크박스 삭제 — RT 7일 고정, MVP 단순화
2. 체크박스 유지 → 백엔드 rememberMe 파라미터 추가 — 체크 시 장기(7~30일), 미체크 시 단기(세션/수시간)

**Why:** 유저 선택이 토큰 만료 정책에 영향을 주는 사안이라 PM/백엔드 합의 필요.

## 안건 2: 치료사 인증 입력 임시 저장 (FNC-004) 필요성

현재 치료사 인증은 단일 폼 1개 화면 (면허번호 + 치료영역 선택 + 증빙 이미지). 이미지는 보안상 브라우저가 자동 복원 불가 → 다시 선택 필요.

**논의 포인트:**
- 단일 화면 폼에서 임시 저장이 실질적으로 필요한가?
- MVP에서 빼도 되는지, 아니면 UX상 필수인지
- 현재 P2로 잡혀 있음

## 안건 3: Google 소셜 로그인 (FNC-006) MVP 포함 여부

3/25에 회원가입+치료사인증 통합 플로우 전환으로 삭제한 상태. 현재 명세는 MVP+로 표기 중.

**논의 포인트:**
- MVP에 다시 넣을지, Post-MVP로 확정할지
- 넣는다면 백엔드 OAuth 작업 범위 확인 필요

**How to apply:** 4/7~8 주간회의 이야기 나오면 세 안건 모두 알려줄 것. 결정 후 FNC-002, FNC-004, FNC-006 명세 업데이트.
