대규모 마이그레이션/리팩토링 가이드를 요청받으면 **단위별 단계로 쪼개서**, 각 단계 종료 시점에 **앱이 동작하는 상태**를 유지하는 체크포인트를 끼워 제공할 것.

**Why:** 2026-04-22 R-01 React Query 마이그레이션 가이드 중 "적용 범위를 단위별로 잘 쪼개서 막막함이 안들도록 잘 가이드해줘" 요청. 523줄짜리 파일을 한 번에 바꾸려 하면 막막함이 작업 동기를 깎음. 또한 싱글태스크 선호(`user_work_focus_pattern.md`)와 맞닿아 있음.

**How to apply:**
- 교체 대상이 여러 개(예: 탭 3개)면 **1개 먼저 교체 → 검증 → 다음 하나** 순서
- 나머지 대상은 기존 코드를 **비교 대상**으로 그대로 남김 (한 번에 다 지우지 않음)
- 각 단계 종료 기준 명시: `tsc` 통과 + 브라우저 기능 동작 + 구체 시나리오(페이지 이동/재시도/탭 왕복 등) 확인
- 각 단계에 대략 예상 소요 시간 제공 → 사용자가 멈출 지점을 예측 가능
- 중간에 세션이 끝나도 그 시점까지 앱이 동작하면 안전한 상태로 마무리 가능

**적용 예시 (2026-04-22 ProfilePage RQ 마이그레이션 5단계 분할):**
1. import + posts 탭만 교체 + 재매핑 + onRetry 수정 (25분)
2. commented 탭 교체 (10분)
3. scrapped 탭 교체 (10분)
4. 정리 — 불필요한 타입 import 제거 (10분)
5. 브라우저 실제 시나리오 검증 (10분)
