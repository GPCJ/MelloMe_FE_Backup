## 상황 (04-04)
- 백엔드 개발자가 응답 형식을 전체적으로 통일하는 리팩토링 진행 중
- 페이지네이션 응답: `posts` → `items`로 변경 등
- 프론트가 맞춰 수정해도 다시 뒤집힐 수 있는 상태

## 결정
- 프론트-백엔드 연동 작업 일시 중단
- 백엔드 응답 확정 후 재개

## 검토한 대안: API 레이어 변환
- `api/*.ts` 파일에서 백엔드 응답 → 프론트 내부 형식으로 변환
- 백엔드가 바뀌면 API 파일만 수정, 컴포넌트는 그대로
- 미적용 상태 — 백엔드 확정 후 적용 여부 재검토

**Why:** 움직이는 과녁에 맞추는 것은 시간 낭비
**How to apply:** 백엔드 응답 확정 확인 후 연동 작업 재개. API 레이어 변환 패턴 도입 검토
