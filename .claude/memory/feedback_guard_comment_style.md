레이스 컨디션 가드, workaround, 임시 대응 같은 **비자명한 수정 코드**에는:
- 한국어로 "왜 이 코드가 필요한지"를 **상세 주석**으로 달 것.
- 근본 원인이 다른 임시 대응에 의존하면(예: `T1`, `B-01` 같은 태그) 그 의존 관계를 주석에 명시해, 임시 대응 해제 시 함께 제거할 단서를 남길 것.
- 주석 한 줄이 아니라 필요하다면 여러 줄 써도 됨 (일반 CLAUDE.md의 "WHY만 짧게" 가이드라인을 예외적으로 덮어쓴다).

**Why:** 2026-04-20 프로필 편집 HIGH 레이스 가드 조치 시 사용자가 "HIGH 코드 생성하면서 주석 꼬박꼬박 알기쉽게 적어줘"라고 명시 요청. 임시 대응 코드는 시간이 지나면 왜 있는지 잊혀지고 잘못 제거되기 쉬워서 **맥락 보존 가치**가 높다고 판단.

**How to apply:**
- 다음 상황에서 상세 주석을 기본으로 작성한다:
  - 레이스 컨디션 가드 (이중 가드, 순서 의존 로직)
  - 백엔드/외부 시스템의 버그·미구현을 우회하는 프론트 로직
  - 특정 환경변수/인프라 이슈 대응 (예: `resolveImageUrl`의 localhost 치환)
- 일반 비즈니스 로직이나 자명한 코드에는 여전히 "주석 최소화" 원칙 적용 — 예외는 위 3종에 한함.

## 추가 규칙 — backlog ID 인라인 참조 (2026-04-22)

backlog 항목(B-xx, T-xx 등)이 존재하고 그 항목 해소와 함께 제거/수정이 예정된 **호출부**에는 **인라인 참조 한 줄**을 달 것.

- **형식**: `(B-07 presigned URL 전환 완료 시 호출 제거 검토)` 같이 backlog ID + 해소 조건 명시.
- **기존 주석이 있으면 그 블록 끝에 한 줄 추가**(새 블록 분리 금지) — 제거 시 블록 단위로 한 번에 처리 가능.
- **예시**: `PostDetailPage.tsx:322` — `resolveImageUrl(img.imageUrl)` 호출부에 B-07 참조 추가.

**Why (2026-04-22 사용자 확인):** 해당 라인을 편집하는 사람 입장에서 인라인 주석 조회가 메모리/백로그 조회보다 즉시적이고 빠름. 메모리/백로그는 전체 조망용, 주석은 로컬 포인터 — 두 층은 보완 관계. 내가 처음에 "굳이 안 달아도 됨"으로 판단했으나 사용자가 접근 속도 근거로 오버라이드.

**How to apply:** backlog에 연동된 임시/조건부 호출부 편집 시, 기존 주석 블록 말미에 `(B-xx ... 시 ...)` 한 줄을 표준으로 추가. 기본 스탠스("주석 최소화")를 이 경우에 한해 완화.
