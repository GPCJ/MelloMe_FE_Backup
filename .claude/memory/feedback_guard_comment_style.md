레이스 컨디션 가드, workaround, 임시 대응 같은 **비자명한 수정 코드**에는:
- 한국어로 "왜 이 코드가 필요한지"를 **상세 주석**으로 달 것.
- 근본 원인이 다른 임시 대응에 의존하면(예: `T1`, `B-01` 같은 태그) 그 의존 관계를 주석에 명시해, 임시 대응 해제 시 함께 제거할 단서를 남길 것.
- 주석 한 줄이 아니라 필요하다면 여러 줄 써도 됨 (일반 CLAUDE.md의 "WHY만 짧게" 가이드라인을 예외적으로 덮어쓴다).

**Why:** 2026-04-20 프로필 편집 HIGH 레이스 가드 조치 시 사용자가 "HIGH 코드 생성하면서 주석 꼬박꼬박 알기쉽게 적어줘"라고 명시 요청. 임시 대응 코드는 시간이 지나면 왜 있는지 잊혀지고 잘못 제거되기 쉬워서 **맥락 보존 가치**가 높다고 판단.

**How to apply:**
- 다음 상황에서 상세 주석을 기본으로 작성한다:
  - 레이스 컨디션 가드 (이중 가드, 순서 의존 로직)
  - 백엔드/외부 시스템의 버그·미구현을 우회하는 프론트 로직
  - 특정 환경변수/인프라 이슈 대응 (예: `resolveImageUrl`의 localhost 치환)
- 일반 비즈니스 로직이나 자명한 코드에는 여전히 "주석 최소화" 원칙 적용 — 예외는 위 3종에 한함.
