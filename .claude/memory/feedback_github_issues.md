개선사항/기술부채는 GitHub Issues로 기록.

**Why:** 코드 TODO 주석 + 이슈 트래커 연동이 실무 표준. 머리로 기억하지 않고 시스템에 남기기 위함.

**How to apply:**
- 사용자가 "이슈 기록해줘" 요청 시에도 먼저 **이슈로 작성할 만한 내용인지, 주석만으로 충분한지** 판단할 것.
- 확인 안 된 추정 기반 내용은 이슈 부적절 → 주석으로 대응, 사실 확인 후 이슈화.
- 검토 중 발견한 개선사항을 자동으로 이슈 생성하지 말 것 — 반드시 요청 시에만.
