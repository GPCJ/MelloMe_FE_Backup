백엔드 개발자가 AI 제안을 검토 없이 바로 적용하는 경향 → 프론트 입장에서 급방향 전환 발생.

**Why:** 구현 방법까지 제시하면 AI가 다른 방향 제안 시 급커브. 기술적 디테일은 혼란만 줌.
**How to apply:**
- "왜 문제인지" + "기대하는 결과"만 전달, 구현 방식은 백엔드에 위임
- 스크린샷/영상으로 유저가 겪는 UX 보여주기
- 이슈보다 디스코드 직접 전달 선호
- 백엔드 개발자가 LLM에 복붙할 것을 전제로, LLM이 맥락을 정확히 이해할 수 있는 형태로 텍스트 작성
