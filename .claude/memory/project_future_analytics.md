## 학습 완료 사항 (2026-04-02)

### 왜 프론트에서 수집하나?
- 백엔드는 API 호출만 알 수 있고, 클릭/스크롤/체류시간/이탈 같은 브라우저 내 행동은 프론트에서만 감지 가능
- 팀원들도 "행동 데이터 수집은 프론트에서 해야 한다"고 합의

### 추천 도구 조합 (무료)
- **GA4 (Google Analytics 4)**: 페이지뷰, 이탈률, 세션 시간, 유입 경로 자동 수집 + 커스텀 이벤트
- **Microsoft Clarity**: 히트맵(클릭 위치 시각화), 세션 녹화, rage click/dead click 감지
- 둘 다 무료, 백엔드 변경 없음

### 설치 방법
1. `index.html`에 GA4 + Clarity 스크립트 삽입 (각 1줄)
2. React SPA이므로 라우트 변경을 GA4에 알려주는 커스텀 훅 필요 (`useLocation` + `gtag('event', 'page_view')`)
3. Clarity는 SPA 라우트 변경 자동 감지 → 별도 처리 불필요

### 도입 단계
1. **1단계**: GA4 + Clarity 설치 (30분, 프론트만)
2. **2단계**: 핵심 이벤트 5~10개 정의 (팀 논의) → 컴포넌트에 `gtag()` 호출 삽입
3. **3단계**: 2~4주 데이터 쌓인 후 패턴 분석 → 개선

### 이벤트 설계 원칙
- 이름: `동사_목적어` 형태 (예: `sign_up_complete`, `post_create`, `comment_create`)
- 파라미터로 맥락 추가 (예: `therapy_area`, `has_image`, `content_length`)
- "뭘 알고 싶은지" 질문 먼저 정의 → 그에 맞는 이벤트 설계

### 멜로미에서 측정할 만한 질문 후보
- 가입 → 치료사 인증 전환율
- 글 읽기만 하는 사람 vs 글 쓰는 사람 비율
- 치료영역별 게시글 인기도
- 어디서 이탈하는지

**Why:** MVP 이후 서비스 개선을 위해 필요하지만, 현재 단계에서 도입하면 복잡도만 증가
**How to apply:** MVP 완료 후 1단계(도구 설치)부터 시작. 이 메모리를 꺼내서 바로 진행 가능
