Notion TIL은 그대로 유지하면서 Velog에 선별 공유. 블로그를 "독립 스트림"이 아닌 **"Notion 자산의 2차 가공"**으로 본다.

**Why:**
- 매일 두 군데 쓰는 건 지속 불가 → 리소스 분산 걱정 해소
- Notion만 쓰면 취업 시 개발 활동 아카이브 접근성 떨어짐 (외부 피드백)
- 의무 기반 꾸준함은 반년 내 꺾이지만, 자랑/보람 기반은 에너지가 스스로 생성됨

**How to apply:**

1. **플랫폼**: Velog (한국 개발자 유입 + 마크다운 네이티브 + 이식성)
2. **빈도**: 주 **최소** 1회. 상한 없음(더 써도 OK). 작성 시간 자유 — 고정 세션 강요 금지
3. **승격 룰 (OR 조건)**:
   - ① 삽질 1시간 이상 트러블슈팅
   - ② 다른 사람도 겪을 법한 문제
   - ③ 내가 한 달 후 다시 참고할 것
   - ④ **나만 알기 아까움 / 자랑하고 싶음 ← 최우선 트리거**

   ④ 해당 시 ①②③ 안 맞아도 무조건 후보. "쓰고 싶다"는 감정 자체가 가장 신뢰할 수 있는 품질 신호.
4. **실패 안전장치**: 한 주 스킵 OK → 2주 1회로 자동 완화. **죄책감 루프 차단이 최우선**(0 → 자책 → 포기 흐름 방지)
5. **초기 3개월 제약**: 신규 창작 금지, **Notion 자산 재가공만**. 제로부터 쓰면 편당 3시간 → 지속 불가
6. **시리즈 활용**: Velog Series 기능으로 2부작 가능 (예: "직접 구현기" → "라이브러리 선택 후기")

**운영 팁:**
- Notion TIL에 `⭐블로그후보` 태그로 승격 후보 예약 → 판단 비용 제거
- 폐기 예정 코드여도 "왜 다음 선택을 했는지" 서사로 활용 가능

**취업용 관점 보정:**
- "취업용이니까 객관적으로 맞춰야 한다" ❌ → 오히려 **선명한 주관**이 유리
- 이유: 인사담당자/면접관은 "글쓴이가 보이는 글"을 기억함. 평균치로 다듬은 글은 지원서 100개 중 하나. 또 면접에서 "왜 이렇게 썼어요?"에 방어하려면 본인 주관이어야 답변 가능.
- 남 시선 검토는 발행 직전 **한 번만**. 쓰는 단계에선 죽이지 않음.
