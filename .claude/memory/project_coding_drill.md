AI 코드 생성 의존도가 높아져 실제 코딩 역량에 대한 불안 해소를 위해 도입.

**구조**: 읽기(15분) → 설명(10분) → AI 없이 구현(45~60분) → 비교(10~15분) → 기록(30초)

**파일**:
- `docs/coding-drill-guide.md` — 루틴 가이드 + 난이도별 파일 풀
- `docs/coding-drill-log.md` — 매일 3줄 수동 기록 (도전/막힌곳/수준)
- `/update-drill` — 주간 노션 업로드 커맨드

**1주차**: 진단 테스트 (ProtectedRoute → WelcomePage → LandingPage → LoginPage → useAuthStore)

**기록 방식**: 매일 로컬 마크다운에 수동 기록, 주 1회 노션 배치 업로드

**Why:** 사용자가 "내가 실제로 이 수준의 역량을 가지고 있는가?" 불안을 느낌. 별도 프로젝트가 아닌 멜로미 코드 자체를 교재로 활용.

**How to apply:** 코딩 드릴 관련 대화 시 참고. 루틴 진행 상황이나 난이도 조정 요청 시 guide/log 파일 확인.
