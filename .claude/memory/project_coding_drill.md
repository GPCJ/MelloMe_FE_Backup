## 현재 버전: v2 (설계 설명 연습) — 2026-04-21 전환

v1(백지 재구현)은 종료. 이유:
- 프론트 전체 코드 학습 완료 (wiki `19`)
- 실전 트러블슈팅은 이미 AI 없이 주도
- 04-17 판단: "RQ 이전 로직 깊이 파는 건 유지보수자에게나 유효" — 재구현형 학습에 회의 이미 표명
- 남은 약점은 타이핑이 아니라 **설계 언어화** (`user_dev_style.md`)

## v2 구조 (주 2~3회 × 30분)

1. **대상 선정 (2분)**: 최근 1주 내 작업한 파일 1개, hot paths 우선
2. **자문 (10분)**: 질문 풀에서 2~3개 선택
   - 경계/책임: 왜 분리, 어디까지 책임
   - 상태/데이터: 왜 이 위치, props vs context vs store
   - 의존/구조: 누가 의존, 폴더 경계 이유
   - 면접 모의: 1분 설명, 트레이드오프
3. **답변 작성 (15분)**: 면접 답변 톤으로 서술, 말하듯이
4. **기록 (3분)**: "질문 / 답변 요점 / 막힌 부분" 3줄

## Why
역량 불안의 실체는 "코드 타이핑"이 아니라 "설계/아키텍처 언어화"(`user_dev_style.md`). v1은 전자만 훈련했음. v2는 실전 코드와 면접 대비를 동시 타격.

## How to apply
- "드릴"이라고 부르면 v2 루틴 안내
- 대상 파일 추천 요청 시 hot paths(PostEditPage, useFileAttachment, PostCreatePage) 우선 제안
- v1 파일 풀(초급~최상급 줄 수 기반)은 더 이상 사용 안 함
