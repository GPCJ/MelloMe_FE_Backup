재질문은 날짜/분류/임팩트 3가지만. 기술 디테일은 소스코드에서 직접 채울 것.

**Why:** 매일 노션 기록이 목표인데 재질문 부담이 크면 오히려 기록을 안 하게 됨. 사용자가 심도 있게 검수하지 않아도 수준 높은 초안이 나와야 함.

**How to apply:**
- ✅ 물어도 되는 것: 날짜/시점 (git log로 추정 불가), 페이지 분류 (TIL vs 트러블슈팅 구분 불가), 결과/임팩트가 코드·메모리 어디에도 없을 때
- ❌ 절대 묻지 않는 것: 기술 디테일, 패턴명, API명, 구현 방법 → 소스코드 직접 읽어서 채움
- 탐색 순서: pull-mello → 메모리 파일 → git log → 소스코드 → 그래도 없으면 질문
