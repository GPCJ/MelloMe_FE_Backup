노션에 기록할 작업/학습 내용을 최소한의 사용자 입력으로 수준 높은 초안으로 뽑아서 노션에 작성해줘.

## 흐름

### 1단계 — 맥락 수집
사용자에게 딱 한 가지만 물어봐:
> "어떤 작업이나 학습을 기록하고 싶어? (날짜, 주제 한 마디면 충분해)"

### 2단계 — 자동 리서치 (사용자 개입 없이)
아래 순서대로 정보를 최대한 긁어와서 초안을 채워:

1. `./scripts/memory-sync.sh pull-mello` 실행해서 최신 메모리 동기화
2. `.claude/memory/MEMORY.md` 인덱스 보고 관련 메모리 파일 탐색·읽기
3. `git log --oneline -20` 으로 관련 커밋 확인 (날짜, 파일 변경 내역)
4. 관련 소스코드 파일 직접 읽기 (구현 디테일, 패턴, 타입 등 보충)
5. 위 4단계로도 채워지지 않는 항목이 있을 때만 사용자에게 질문

### 3단계 — 초안 분류 및 작성
내용을 분석해서 빌더스 리그 서브페이지별로 분류:
- **TIL**: 오늘 배운 개념·패턴·기술적 학습 → `323c8200749b80c2bbe6caf194055593`
- **🔧 트러블슈팅**: 마주친 문제와 해결 과정 → `322c8200749b81f39f71f9c8a4d6eb44`
- **🏗 설계 결정 & 아키텍처**: 의식적으로 내린 정책/구조 결정 → `32dc8200749b81e899bde7aea0a37937`
- **📈 프로젝트 성과 & 지표**: 수치/기능 단위 성과, 배포 완료, UX 개선 → `32dc8200749b8157b695e5e84e60e01b`

각 페이지 컨벤션에 맞춰 초안 작성:
- **TIL**: 제목 `YYYY-MM-DD — [주제]` / `**분류**:` → `## 오늘 한 것` → `## 배운 것 / 인사이트` → `## 포트폴리오 어필 포인트`
- **트러블슈팅**: 제목 `#NNN — [문제]` (기존 목록 마지막 번호 +1) / `**날짜**:` `**분류**:` `**난이도**:` → `## 문제 상황` → `## 원인 분석` → `## 해결 과정` → `## 핵심 개념` → `## 면접 포인트`
- **설계 결정**: 제목 `MM-DD — [결정 주제]` / `## N. [결정 제목]` → `**문제:**` → `**검토한 선택지:**` → `**결정:**` → 근거 → `---`
- **성과 & 지표**: 제목 `MM-DD — [주제] 성과` / 기능별 bullet (배포 상태, UX 개선, 안전성) + 이력서 bullet 예시

초안을 사용자에게 보여주기 전에 분류 계획을 먼저 안내해:
> 예: "TIL 1개 + 설계 결정 1개로 나눌 예정이야. 초안 확인해봐."

초안을 사용자에게 보여주고 승인을 받아.

### 4단계 — notion_draft.md 기존 내용 확인
승인 전에 `.claude/memory/notion_draft.md`를 읽어서 기존 내용이 있는지 확인:
- **기존 내용 없음**: 바로 저장 진행
- **기존 내용 있음**: 내용을 짧게 요약해서 사용자에게 전달
  > 예: "기존 초안에 '04-13 무한 스크롤 TIL'이 있어. 노션에 이미 작성된 내용이야?"
  - 이미 작성됨: 기존 내용 덮어쓰고 새 초안만 저장
  - 아직 미작성: 기존 내용 + 새 초안을 합쳐서 저장

### 5단계 — 저장 및 노션 작성 분기
1. 초안을 `.claude/memory/notion_draft.md`에 저장
2. 사용자에게 질문:
   > "지금 이 세션에서 바로 노션에 쓸까, 아니면 나중에 노션 권한 있는 계정에서 `/post-notion-draft`로 쓸까?"
3. **지금 쓰기**: 분류된 각 페이지에 notion-create-pages로 서브페이지 생성
4. **나중에 쓰기**: 메모리 저장 완료 안내 후 종료
5. 어느 분기든 완료 후 사용자에게 질문:
   > "백업 레포에 메모리 동기화할까? (`push-mello` 실행)"
   - 승인 시: `./scripts/memory-sync.sh push-mello` 실행

## 재질문 기준
아래 항목을 메모리·git·소스코드로도 채울 수 없을 때만 질문:
- 날짜/시점 (git log로 추정 불가할 때)
- 페이지 분류 (TIL인지 트러블슈팅인지 구분 불가할 때)
- 결과/임팩트가 코드상에도 메모리에도 없을 때

**절대 묻지 않아도 되는 것**: 기술 디테일, 패턴명, API명 → 소스코드에서 직접 읽어서 채울 것

## 주의사항
- 트러블슈팅 번호는 기존 페이지 마지막 번호 확인 후 이어서 부여
- 컨벤션 확인이 필요하면 해당 페이지의 기존 서브페이지를 notion-fetch로 열람
- 하나의 작업이 여러 페이지에 해당하면 페이지별로 나눠서 각각 서브페이지 생성
