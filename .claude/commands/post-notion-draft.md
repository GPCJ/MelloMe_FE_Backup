`.claude/memory/notion_draft.md`에 저장된 초안을 노션에 작성해줘.

## 작업 순서

1. 먼저 `.claude/memory/notion_draft.md`를 읽어서 현재 내용을 확인하고, 사용자에게 "pull-mello로 최신 메모리를 먼저 가져올까요, 아니면 현재 초안으로 바로 노션 작성을 진행할까요?"라고 질문. 사용자가 pull을 선택한 경우에만 `./scripts/memory-sync.sh pull-mello` 실행 후 draft 재읽기
2. 확정된 `.claude/memory/notion_draft.md` 내용으로 진행
3. 초안 내용을 분석해서 빌더스 리그 서브페이지별로 분류:
   - **TIL**: 오늘 배운 개념·패턴·기술적 학습 → `323c8200749b80c2bbe6caf194055593`
   - **🔧 트러블슈팅**: 마주친 문제와 해결 과정 → `322c8200749b81f39f71f9c8a4d6eb44`
   - **🏗 설계 결정 & 아키텍처**: 의식적으로 내린 정책/구조 결정 → `32dc8200749b81e899bde7aea0a37937`
   - **📈 프로젝트 성과 & 지표**: 수치/기능 단위 성과, 배포 완료, UX 개선 → `32dc8200749b8157b695e5e84e60e01b`
4. 각 페이지의 **기존 컨벤션**에 맞춰 서브페이지 생성:
   - **TIL**: 제목 `YYYY-MM-DD — [주제]` / `**분류**:` → `## 오늘 한 것` → `## 배운 것 / 인사이트` → `## 포트폴리오 어필 포인트`
   - **트러블슈팅**: 제목 `#NNN — [문제]` (기존 목록 마지막 번호 +1) / `**날짜**:` `**분류**:` `**난이도**:` → `## 문제 상황` → `## 원인 분석` → `## 해결 과정` → `## 핵심 개념` → `## 면접 포인트`
   - **설계 결정**: 제목 `MM-DD — [결정 주제]` / `## N. [결정 제목]` → `**문제:**` → `**검토한 선택지:**` → `**결정:**` → 근거 → `---`
   - **성과 & 지표**: 제목 `MM-DD — [주제] 성과` / 기능별 bullet (배포 상태, UX 개선, 안전성) + 이력서 bullet 예시
5. 완료 후 생성된 노션 페이지 URL 목록 안내

## 주의사항

- `notion_draft.md`가 비어있거나 업로드 대기 내용이 없으면 사용자에게 알릴 것
- 컨벤션 확인이 필요하면 각 페이지의 기존 서브페이지를 notion-fetch로 열람 후 맞출 것
- 트러블슈팅 번호는 기존 목록의 마지막 번호를 확인 후 이어서 부여할 것
- 하나의 작업이 여러 페이지에 해당하면 페이지별로 나눠서 각각 서브페이지 생성
- 노션 작성 후 `notion_draft.md`를 초기화할지 사용자에게 물어볼 것
