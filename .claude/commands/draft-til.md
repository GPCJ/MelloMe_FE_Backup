이번 대화 내용과 메모리를 바탕으로 TIL + 빌더스 리그 초안을 작성하고 메모리에 저장해줘.

## 작업 순서

1. 이번 대화에서 작업한 내용, 배운 것, 해결한 문제 정리
2. 아래 양식으로 초안을 채팅창에 먼저 출력
3. 사용자 확인 후 승인이 오면 `.claude/memory/til_draft.md`에 저장 (기존 내용 덮어쓰기)
4. 저장 완료 후 안내: "다른 기기에서 `/pull-mello` 후 `til_draft.md` 확인하세요"

## 판단 기준

**TIL 후보:**
- 대화 중 새로 배운 개념, 도구, 패턴
- 문제를 해결하면서 알게 된 원인/원리
- 이전에 몰랐던 동작 방식이나 제약 사항

**빌더스 리그 후보:**
- 면접에서 "이런 경험 있습니다"로 말할 수 있는 내용
- 문제 해결 과정이 구체적으로 설명 가능한 경험
- 기술 선택/설계에서 트레이드오프를 고려한 판단

오늘 TIL이 없다면 솔직하게 "오늘은 새로 배운 내용이 없습니다" 라고 알려줘.
빌더스 리그 해당 카테고리가 없으면 해당 섹션은 생략.
단순 반복 작업(push, 파일 정리 등)은 생략하고 의미 있는 작업만 기록.

## 작성 원칙

- 나 자신이 나중에 읽었을 때 바로 이해할 수 있게 작성
- 기술 용어는 그대로 써도 되지만 맥락 설명 필수
- 트러블슈팅이 있었다면 배운 핵심 원리 위주로 요약
