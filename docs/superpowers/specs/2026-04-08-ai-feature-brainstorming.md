# AI 기능 브레인스토밍 — 2026-04-08

## 배경

멜로미 서비스의 가장 큰 리스크는 **콜드 스타트** — 초기에 볼 만한 콘텐츠가 부족하면 유저가 이탈하는 악순환. AI 기능이 이걸 깨는 데 기여할 수 있다.

## 핵심 결정 사항

| 항목 | 결정 |
|------|------|
| 대상 | 인증 치료사(THERAPIST) + 일반 유저(USER) |
| 핵심 페인포인트 | 고립감, 정보 탐색, 전문 지식 접근 |
| 최우선 시나리오 | "나만 이런 경험?" → "이 아이에게 맞는 접근법?" |
| 데이터 민감도 | 본문 분석 기반 (정책 제약 시 태그 기반 fallback) |
| 답변 출처 | 커뮤니티 경험 메인 + 외부 문헌은 참고 출처 |
| UI 형태 | 게시글 작성 시 자연스러운 보조 |
| 성공 기준 | 게시글 작성률 증가 (콜드 스타트 해결) |
| 기술 구현 | 기능 우선, 기술은 그에 맞춰 결정 |

## 제안 방향

### 메인: 글쓰기 AI 어시스턴트

게시글 작성 과정에서 AI가 비침습적으로 작성을 돕는 기능. 핵심 원칙은 "글쓰기를 방해하지 않으면서, 혼자가 아니라는 확신을 준다".

**트리거 플로우:**

1. 치료 영역 + 연령대 선택 시 → 매칭되는 기존 게시글 수 표시
2. 본문 100자+ 작성 시 → "비슷한 경험을 공유한 글이 있어요" 배너 (유사 사례 2-3개)
3. 발행 시 → "이 주제에 관심 있는 치료사 N명에게 도달할 수 있어요"

**콜드 스타트 대응:**
- 글 0건 → 표시하지 않음
- 유사 글 없음 → "이 주제의 첫 사례가 될 수 있어요"로 전환
- 유저 수 적음 → "관심 있는 치료사들에게 알림을 보낼게요"

### 보조: AI 초기 응답자

게시글 발행 후 AI가 첫 번째 반응을 제공.

- 관련 사례 요약 + 참고 자료를 자동 제공
- 인간 댓글이 달리면 자연스럽게 접힘
- "글을 써도 반응이 없다"는 허탈감 방지

### 후순위: 사례 탐색기 (콘텐츠 축적 후)

별도 탐색 공간에서 "내 상황과 비슷한 사례 찾기". 콘텐츠가 충분히 쌓인 후 도입.

## PM 커피챗 — 논의 주제 & 질문

### 콜드 스타트 전략 합의

- "초기 유저가 10명일 때, AI 추천이 '비슷한 글 0건'이면 오히려 역효과라고 생각하는데 — PM 관점에서 **최소 몇 개의 게시글이 쌓여야 AI 기능을 켜도 된다**고 보시나요?"
- "콘텐츠가 부족한 초기에 **'이 주제의 첫 사례가 되어보세요'** 같은 메시지로 전환하는 건 어떻게 보시나요? 동기부여가 될까요, 압박이 될까요?"

### 데이터 민감도 정책

- "치료사들이 올리는 사례에 아동 정보가 포함될 수 있는데, AI가 **본문을 분석해서 유사 사례를 매칭**하는 것에 대해 유저들이 어떻게 반응할 것 같으시나요?"
- "개인정보처리방침이나 이용약관에 **'AI 분석에 활용됩니다'를 명시**해야 할 텐데, 이 부분의 타이밍과 우선순위는?"

### AI 초기 응답자 (보조 기능)

- "글을 쓴 후 아무 반응이 없는 경험이 이탈의 가장 큰 원인일 수 있는데 — AI가 **첫 번째 반응(관련 사례 요약)을 자동으로 달아주는 것**에 대해 어떻게 생각하시나요?"
- "'봇이 댓글 단다'는 느낌을 피하려면 **어떤 톤과 형태**가 적절할까요? 댓글? 카드? 별도 섹션?"

### 성공 지표

- "AI 기능의 성공을 **게시글 작성률 증가**로 보고 있는데, PM 관점에서 동의하시나요? 다른 우선순위 지표가 있으시나요?"
- "측정 가능한 목표치를 잡는다면 — 예를 들어 **AI 보조 사용 시 글 작성 완료율 X% 향상** 같은 기준이 현실적일까요?"
