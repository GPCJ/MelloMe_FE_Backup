# 홈 피드 화면 리디자인 스펙

## 개요

피그마 확정 디자인 기반으로 홈 피드(PostListPage) UI를 리디자인한다.
이번 작업은 **UI만 구현**하며, 백엔드 미구현 기능의 로직은 빈 상태로 두고 별도 기록한다.

## 변경 범위

### 1. 탭 시스템 (신규)

- **전체 피드** (active, 하단 밑줄) | **팔로우**
- 전체 피드: 기존 게시글 목록 표시
- 팔로우: 빈 상태 UI ("팔로우한 치료사의 글이 여기에 표시됩니다" 등)
- 탭 전환은 로컬 state로 관리

### 2. 필터칩

- 기존 유지: 전체 / 작업치료 / 언어치료 / 놀이치료 / 인지치료
- 스타일 변경: 선택된 칩은 검정 filled, 나머지는 outline
- 가로 스크롤 (overflow-x-auto)

### 3. PostCard 리디자인 (핵심)

기존 카드형 그리드 → 리스트형 피드로 변경.

**구조 (위→아래):**

| 행  | 내용                                                                                                          |
| --- | ------------------------------------------------------------------------------------------------------------- |
| 1   | 프로필 이미지(원형, 이니셜 fallback) + 닉네임 + 인증뱃지(미인증/인증완료) + 상대 시간 + 우측 끝 북마크 아이콘 |
| 2   | 해시태그 칩들 (#언어치료 #도움되요) — outline 둥근 칩, 목데이터                                               |
| 3   | 본문 미리보기 텍스트 (2-3줄 truncate)                                                                         |
| 4   | 📎 첨부파일 있음 (첨부파일 있을 때만 표시)                                                                    |
| 5   | 댓글 수 + 조회수                                                                                              |

**블러 게시글 (인증 전용):**

- 1행, 2행 동일
- 3행: 회색 배경 박스 "🔒 인증된 회원에게만 공개된 게시물입니다."
- 4행: 첨부파일 표시 (있으면)
- 5행: 댓글 수 + 조회수 (동일)

### 4. 헤더

- "치료사 커뮤니티" 타이틀 + 우측 검색 아이콘
- 검색 아이콘은 UI만 (클릭 동작 없음)

### 5. 하단 네비

- 기존 유지 (홈 / 글쓰기 / 프로필)

## UI만 구현 — 나중에 채울 로직

| 항목                      | 설명                              | 의존성               |
| ------------------------- | --------------------------------- | -------------------- |
| 팔로우 탭 데이터          | 팔로우한 유저의 게시글 fetch      | 백엔드 팔로우 API    |
| 스크랩 토글               | 북마크 아이콘 클릭 시 스크랩/해제 | 백엔드 스크랩 API    |
| 블러/언블러 로직          | 인증 상태 기반 본문 표시/숨김     | 백엔드 postType 필드 |
| 해시태그 데이터           | 실제 태그 데이터 연동             | 백엔드 태그 필드     |
| therapyArea 필터 파라미터 | 필터칩 선택 시 API 호출           | 백엔드 쿼리 파라미터 |
| 검색 기능                 | 검색 아이콘 클릭 → 검색 화면      | 검색 API + UI        |

## 기술 결정

- PostCard를 **별도 컴포넌트로 분리** (현재 PostListPage 내 인라인)
- shadcn/ui 컴포넌트 기반
- 아이콘: lucide-react (기존 사용 중)
- 레이아웃: 리스트형 단일 컬럼
- 해시태그: 목데이터 배열로 PostCard에 전달
